JSON to JSON-LD

Changing JSON into JSON-LD is very easy! All that needs to be done is implementing the URIs which make the data linked!

You need an @context tag which refers to the datex vocabulary (e.g. vocab.datex.org/terms#). On vocab.datex.org you can find all the terms that are defined in Datex II. Once that's done you need an @type tag to indicate what kind of content you're defining. For the parking context it is: "parkingSite".

All that rests is to use the existing URIs from the Datex II vocab website to define the properties of your parking (e.g. parkingName)

Underneath there is a neat drag and drop tutorial to help you get started!

Implementing RDFa into HTML

Converting HTML to RDFa doesn't take much effort. All that has to be done is implementing the right URIs

Start by creating a prefix "dtx" to link to the Datex II vocabulary. The vocabulary can be found at vocab.datex.org

All that has to be done now is defining the type of your information (e.g. a parkingsite) with an existing URI on the vocab.datex.org in the typeOf tag. Also give it its own URI linking to the content on your page by using a resource tag on the container where your other data will be in.

Once we've defined the type of information we can give all our other information that defines our parkingsite properties like the name of our parking. You can find all the available properties on vocab.datex.org

Underneath this you'll find our super handy RDFa tutorial. If you do it correctly it will show you a visualization of the dataset!
